### **HTML (20 Questions)**

1. **What is HTML?**  
   HTML (HyperText Markup Language) is used to create the structure of web pages.

2. **What are HTML tags?**  
   Tags are used to define elements in an HTML document.

3. **What is the difference between `<div>` and `<span>`?**  
   `<div>` is a block-level element, and `<span>` is an inline element.

4. **What is the purpose of the `<meta>` tag?**  
   It provides metadata like description, keywords, and author.

5. **What does the `<a>` tag do?**  
   It creates a hyperlink to other web pages or resources.

6. **What is semantic HTML?**  
   Semantic HTML uses meaningful tags like `<article>`, `<section>`, `<header>`.

7. **How do you insert an image in HTML?**  
   Use the `<img>` tag with `src` attribute: `<img src="image.jpg" alt="description">`.

8. **What is the `<form>` tag used for?**  
   It creates a form for user input.

9. **How do you create a list in HTML?**  
   Use `<ul>` for an unordered list and `<ol>` for an ordered list.

10. **What is the difference between the `<thead>` and `<tbody>` tags in a table?**  
    `<thead>` defines the table header, and `<tbody>` defines the table body.

11. **What is the `alt` attribute in an `<img>` tag?**  
    It provides alternative text for the image if it can't be displayed.

12. **How do you create a hyperlink that opens in a new tab?**  
    Add `target="_blank"` to the `<a>` tag.

13. **What is the use of the `<iframe>` tag?**  
    It embeds another webpage inside your page.

14. **What is a DOCTYPE in HTML?**  
    It tells the browser which version of HTML is being used.

15. **What is the `<br>` tag?**  
    It inserts a line break in the content.

16. **How do you make text bold in HTML?**  
    Use the `<strong>` or `<b>` tag.

17. **How do you create a dropdown list?**  
    Use the `<select>` tag with `<option>` tags inside it.

18. **What is the purpose of the `<link>` tag?**  
    It links external resources like stylesheets to the HTML document.

19. **How do you specify the character set in HTML?**  
    Use `<meta charset="UTF-8">` in the `<head>`.

20. **What is an HTML attribute?**  
    An attribute provides additional information about an element, like `href` in `<a>`.

### **CSS (20 Questions)**

1. **What is CSS?**  
   CSS (Cascading Style Sheets) is used to style HTML elements.

2. **How do you apply CSS to an HTML document?**  
   Through inline styles, `<style>` tags, or external stylesheets using `<link>`.

3. **What is the box model?**  
   The box model includes content, padding, border, and margin.

4. **How do you center a block element?**  
   Use `margin: auto;` or Flexbox.

5. **What is Flexbox?**  
   Flexbox is a layout model for arranging elements in rows or columns.

6. **What is the difference between `px`, `em`, and `%` in CSS?**  
   `px` is absolute, `em` is relative to the parent element, `%` is relative to the parent container.

7. **What is a CSS class?**  
   A reusable style that is applied to elements using the `class` attribute.

8. **What is a CSS ID?**  
   A unique style applied to a single element using the `id` attribute.

9. **How do you change the background color in CSS?**  
   Use the `background-color` property.

10. **What is the difference between `visibility: hidden` and `display: none`?**  
    `visibility: hidden` hides the element but keeps its space, `display: none` removes it entirely.

11. **How do you create a responsive layout?**  
    Use media queries, Flexbox, or CSS Grid.

12. **What is a pseudo-class in CSS?**  
    A keyword added to selectors to style elements based on their state, like `:hover`.

13. **How do you style a button on hover?**  
    Use the `:hover` pseudo-class, e.g., `button:hover { background-color: red; }`.

14. **What is CSS specificity?**  
    It determines which CSS rule is applied when multiple rules target the same element.

15. **What is the `z-index` property?**  
    It controls the stack order of elements.

16. **How do you make text italic in CSS?**  
    Use the `font-style: italic;` property.

17. **How do you add a shadow to a box in CSS?**  
    Use the `box-shadow` property.

18. **What is the difference between `inline` and `block` elements?**  
    Inline elements don’t start on a new line, block elements do.

19. **What is the difference between `position: absolute` and `position: relative`?**  
    `absolute` positions an element relative to the nearest positioned ancestor, `relative` moves it relative to its normal position.

20. **How do you hide an element in CSS?**  
    Use `display: none;` or `visibility: hidden;`.

### **JavaScript (40 Questions)**

1. **What is JavaScript?**  
   JavaScript is a programming language used to create dynamic and interactive web content.

2. **What are variables in JavaScript?**  
   Variables store data values, declared using `var`, `let`, or `const`.

3. **What is a function?**  
   A function is a block of code designed to perform a task.

4. **How do you declare a function?**  
   `function myFunction() { ... }`.

5. **What is an array?**  
   An array stores multiple values in a single variable.

6. **How do you loop through an array?**  
   Using `for`, `forEach()`, or `map()` loops.

7. **What is `null` in JavaScript?**  
   `null` represents an intentional absence of any value.

8. **What is `undefined`?**  
   `undefined` means a variable has been declared but has no value assigned.

9. **What are objects in JavaScript?**  
   Objects are collections of key-value pairs, like `{ name: 'John', age: 30 }`.

10. **How do you access an object’s properties?**  
    Using dot notation (`obj.name`) or bracket notation (`obj['name']`).

11. **What is the difference between `==` and `===`?**  
    `==` checks for value equality, `===` checks for value and type equality.

12. **What is an arrow function?**  
    A shorthand way to write functions: `const func = () => { ... }`.

13. **What is the DOM?**  
    The Document Object Model represents the structure of a webpage as a tree.

14. **How do you manipulate the DOM?**  
    Use methods like `getElementById()` or `querySelector()` to select elements and change their properties.

15. **What is an event listener in JavaScript?**  
    It waits for user interactions like clicks, keypresses, etc., and triggers functions.

16. **What is `this` keyword in JavaScript?**  
    `this` refers to the current context in which the function is executed.

17. **What is hoisting?**  
    JavaScript moves variable and function declarations to the top of their scope before executing code.

18. **What is `async/await` in JavaScript?**  
    It’s a way to handle asynchronous operations and promises more easily.

19. **What is a promise?**  
    A promise represents a value that will be available in the future after an asynchronous operation completes.

20. **What is JSON?**  
    JSON (JavaScript Object Notation) is a format for storing and transmitting data objects as text.

21. **What is `localStorage` in JavaScript?**  
    It’s a way to store data on the user's browser that persists even after closing the browser.

22. **What is `sessionStorage`?**  
    It stores data temporarily for the duration of the page session.

23. **How do you convert a JSON string to an object?**  
    Use `JSON.parse()`.

24. **How do you convert an object to a JSON string?**  
    Use `JSON.stringify()`.

25. **What is a callback function?**  
    A function passed as an argument to another function and executed later.

26. **What is a closure in JavaScript?**  
    A closure gives access to an outer function’s variables from within an inner function.

27. **What is the difference between `let` and `var`?**  
    `let` is block-scoped, `var` is function-scoped.

28

. **What is destructuring in JavaScript?**  
    It’s a way to unpack values from arrays or properties from objects into variables.

29. **What is the `spread` operator (`...`)?**  
    It allows you to expand elements of an array or object into separate values.

30. **How do you handle errors in JavaScript?**  
    Using `try`, `catch`, and `finally` blocks.

31. **What is the `fetch` API?**  
    It allows you to make network requests and retrieve data from servers.

32. **What is a regular expression (RegEx)?**  
    A pattern used to match character combinations in strings.

33. **What is the difference between synchronous and asynchronous code?**  
    Synchronous code is executed line by line; asynchronous code doesn’t wait for other operations to finish.

34. **What is the difference between `call()`, `apply()`, and `bind()`?**  
    `call()` and `apply()` immediately invoke a function with a specific `this` value, while `bind()` returns a new function.

35. **What are `map()`, `filter()`, and `reduce()` functions?**  
    - `map()` creates a new array by transforming each element.
    - `filter()` creates a new array with elements that pass a condition.
    - `reduce()` reduces an array to a single value.

36. **What is event bubbling?**  
    Event bubbling means that when an event occurs on an element, it first runs the handler on that element and then on its parent elements.

37. **What is the difference between `innerHTML` and `textContent`?**  
    `innerHTML` includes HTML tags, while `textContent` only includes plain text.

38. **How do you debounce a function in JavaScript?**  
    Debouncing delays the execution of a function to limit how often it runs, often using `setTimeout()`.

39. **What is memoization?**  
    It’s a technique to speed up functions by caching the result of expensive calculations.

40. **What is an Immediately Invoked Function Expression (IIFE)?**  
    A function that runs immediately after it’s defined, like `(function() { ... })()`.

### **React (40 Questions)**

1. **What is React?**  
   React is a JavaScript library for building user interfaces.

2. **What is a component in React?**  
   A component is a reusable piece of UI.

3. **What are props in React?**  
   Props are inputs passed to components to display dynamic data.

4. **What is state in React?**  
   State is an object that holds data that can change over time.

5. **What is JSX?**  
   JSX is a syntax extension of JavaScript that looks similar to HTML and is used in React to describe UI.

6. **What is the difference between functional and class components?**  
   Functional components are simpler, while class components use lifecycle methods and `this`.

7. **What is the use of `useState`?**  
   `useState` is a hook that allows you to add state to functional components.

8. **What is `useEffect` in React?**  
   It’s a hook for performing side effects like fetching data or manipulating the DOM.

9. **What is the virtual DOM?**  
   A lightweight copy of the real DOM that React uses to optimize updates.

10. **How do you create a React app?**  
    Using the `create-react-app` tool.

11. **What is React Router?**  
    It’s a library for adding navigation to React apps.

12. **What is the difference between controlled and uncontrolled components?**  
    Controlled components have their state managed by React, while uncontrolled components rely on the DOM.

13. **What is the `key` prop in React?**  
    `key` helps React identify elements in a list for efficient updates.

14. **What is lifting state up?**  
    Moving state to the closest common ancestor of components that need it.

15. **How do you pass data between components?**  
    By passing props from a parent to a child component.

16. **What is the context API in React?**  
    A way to pass data globally to components without passing props manually.

17. **What is the use of `useContext` hook?**  
    It allows you to use the context API in functional components.

18. **What is a React hook?**  
    Functions that let you use state and other React features in functional components.

19. **How do you handle forms in React?**  
    By using controlled components and handling form data in state.

20. **What is the use of `useRef`?**  
    It’s a hook for accessing and interacting with DOM elements.

21. **What is React Fragment?**  
    A wrapper element (`<Fragment>`) used to group multiple elements without adding extra DOM nodes.

22. **What is `React.memo`?**  
    A higher-order component that optimizes performance by preventing unnecessary re-renders.

23. **What are pure components?**  
    Components that only re-render if their props or state change.

24. **What is the difference between `componentDidMount` and `componentWillUnmount`?**  
    `componentDidMount` runs after a component is rendered, while `componentWillUnmount` runs before it’s removed from the DOM.

25. **How do you optimize React app performance?**  
    By using memoization, lazy loading, and splitting code into smaller chunks.

26. **What is React.lazy?**  
    It’s a way to load components lazily to improve performance.

27. **What is a higher-order component (HOC)?**  
    A function that takes a component and returns a new component with additional functionality.

28. **What is prop drilling?**  
    Passing data through multiple levels of components that don’t need it just to reach a child component.

29. **What is `useReducer`?**  
    A hook for managing complex state logic, often used instead of `useState`.

30. **What is React.StrictMode?**  
    A wrapper component that highlights potential problems in an app during development.

31. **What is the purpose of the `useCallback` hook?**  
    It’s used to memoize a function and prevent it from being recreated on every render.

32. **What is the difference between React and React Native?**  
    React is for building web apps, and React Native is for building mobile apps.

33. **How do you create a custom hook?**  
    By writing a function that uses React hooks and can be reused across components.

34. **What is React's error boundary?**  
    A component that catches JavaScript errors and displays a fallback UI instead of crashing the whole app.

35. **What are controlled forms in React?**  
    Forms where input values are controlled by React’s state.

36. **What is reconciliation in React?**  
    The process React uses to update the DOM by comparing the current state with the virtual DOM.

37. **What is the purpose of `useImperativeHandle`?**  
    It allows you to customize the instance value that is exposed to parent components when using `ref`.

38. **What is lazy loading in React?**  
    A technique to load components or resources only when needed.

39. **How do you handle side effects in React?**  
    Using the `useEffect` hook to handle tasks like fetching data or subscribing to events.

40. **What are portals in React?**  
    A way to render child components outside of their parent component’s DOM hierarchy.

### **Authentication vs. Authorization (10 Questions)**

1. **What is authentication?**  
   Authentication verifies who a user is (e.g., logging in with a username and password).

2. **What is authorization?**  
   Authorization determines what a user can do (e.g., accessing certain pages or resources).

3. **Can you have authentication without authorization?**  
   Yes, a user can be authenticated but not authorized to access all resources.

4. **What is the difference between authentication and authorization?**  
   Authentication confirms identity, while authorization checks permissions.

5. **How is authentication implemented?**  
   Through methods like passwords, biometrics, or OAuth tokens.

6. **How is authorization implemented?**  
   By assigning roles and permissions (e.g., admin, user).

7. **What is multi-factor authentication (MFA)?**  
   A security method that requires more than one form of verification (e.g., password and a code sent to your phone).

8. **What is OAuth?**  
   OAuth is an open standard for authorization that allows third-party apps to access resources on behalf of a user.

9. **What is Single Sign-On (SSO)?**  
   SSO allows users to log in once and gain access to multiple applications.

10. **What is a token in authentication?**  
    A token is a piece of data that verifies the user's identity and is used to access resources.

### **Node.js (15 Questions)**

1. **What is Node.js?**
   - Node.js is a JavaScript runtime built on Chrome's V8 engine that allows you to run JavaScript outside the browser.

2. **What are the advantages of using Node.js for backend development?**
   - Non-blocking, event-driven model, fast performance, large ecosystem of libraries via npm, and ease of building scalable applications.

3. **What is the event-driven architecture in Node.js?**
   - Node.js uses an event-driven model where actions (events) trigger callbacks rather than waiting for blocking operations.

4. **What is the difference between synchronous and asynchronous programming in Node.js?**
   - Synchronous operations block code execution, while asynchronous operations allow other code to run while waiting for a task to finish.

5. **What is `npm` and what is its role in Node.js?**
   - `npm` is the package manager for Node.js, used for managing libraries and dependencies.

6. **What is the purpose of `package.json` in a Node.js project?**
   - It defines the project dependencies, scripts, and metadata (like the version and author of the project).

7. **What is middleware in Node.js?**
   - Middleware functions are used to process requests and responses in an application before they reach the route handler.

8. **How do you handle errors in Node.js?**
   - Using `try...catch` blocks for synchronous code and error-first callbacks or `catch` for promises in asynchronous code.

9. **What is the role of the `fs` module in Node.js?**
   - The `fs` (file system) module allows you to work with the file system, enabling operations like reading, writing, and modifying files.

10. **What is the `cluster` module in Node.js?**
    - The `cluster` module allows you to create child processes that can share the same server port, improving performance on multi-core systems.

11. **What is `buffer` in Node.js?**
    - A buffer is a temporary storage area for data being transferred between two locations in memory, useful when working with streams and binary data.

12. **What are streams in Node.js?**
    - Streams are used to handle reading or writing of data piece by piece, rather than all at once.

13. **What is `process.nextTick()` in Node.js?**
    - It defers the execution of a callback function until the next iteration of the event loop.

14. **What is `Express` in Node.js?**
    - Express is a web application framework for Node.js that simplifies routing and middleware management.

15. **How do you implement authentication in a Node.js application?**
    - By using libraries like `passport.js` or JWT (JSON Web Tokens) to manage authentication and authorization.

### **Express.js (15 Questions)**

1. **What is Express.js?**
   - A minimal and flexible Node.js web framework that simplifies the development of server-side applications.

2. **How do you create a simple Express server?**
   - By importing Express, defining routes, and calling `listen()` on a specific port.

3. **What is routing in Express.js?**
   - Routing determines how an application responds to client requests for specific URLs.

4. **What are query parameters in Express?**
   - Query parameters are appended to URLs after a `?` and are used to send data to the server, e.g., `/users?name=John`.

5. **What is the role of middleware in Express?**
   - Middleware functions execute between the request and response and can modify them, handle errors, or process authentication.

6. **How do you serve static files in Express.js?**
   - By using the `express.static` middleware to serve static files like HTML, CSS, and images.

7. **How do you handle errors in Express.js?**
   - By defining error-handling middleware using a function with four arguments: `err, req, res, next`.

8. **What is the purpose of `body-parser` in Express?**
   - It parses incoming request bodies before your route handlers and makes the data available in `req.body`.

9. **How do you create a route with URL parameters in Express?**
   - By defining a route with `:` to capture parameters, e.g., `/users/:id`.

10. **What is CORS and how do you enable it in Express.js?**
    - CORS (Cross-Origin Resource Sharing) allows restricted resources to be requested from a different domain. It can be enabled using the `cors` package.

11. **How do you set up sessions in Express.js?**
    - Using `express-session` to manage user sessions and persist user data between HTTP requests.

12. **What is the difference between `res.send()` and `res.json()` in Express?**
    - `res.send()` can send any type of response (strings, objects, etc.), while `res.json()` explicitly sends a JSON response.

13. **How do you handle file uploads in Express.js?**
    - Using middleware like `multer` to handle multipart form data and manage file uploads.

14. **How do you set up a proxy in Express?**
    - By using `http-proxy-middleware` to forward requests from your Express app to other services.

15. **How do you use environment variables in an Express app?**
    - By using the `dotenv` package to load environment variables from a `.env` file.

### **MongoDB (15 Questions)**

1. **What is MongoDB?**
   - MongoDB is a NoSQL database that stores data in flexible, JSON-like documents.

2. **What is a collection in MongoDB?**
   - A collection is a group of MongoDB documents, similar to a table in relational databases.

3. **What is a document in MongoDB?**
   - A document is a record in a MongoDB collection, represented as a JSON-like object.

4. **What is the difference between SQL and NoSQL databases?**
   - SQL databases are structured and use schemas, while NoSQL databases like MongoDB are more flexible with unstructured or semi-structured data.

5. **How do you insert a document into a MongoDB collection?**
   - Using the `insertOne()` or `insertMany()` methods.

6. **How do you query documents in MongoDB?**
   - Using methods like `find()`, `findOne()`, and query operators such as `$eq`, `$gt`, etc.

7. **What are indexes in MongoDB?**
   - Indexes improve the speed of query operations by providing efficient ways to look up data.

8. **What is the purpose of the `_id` field in MongoDB?**
   - The `_id` field is a unique identifier for each document in a collection.

9. **How do you update a document in MongoDB?**
   - Using the `updateOne()` or `updateMany()` methods along with operators like `$set` to modify data.

10. **How do you delete a document in MongoDB?**
    - Using `deleteOne()` or `deleteMany()` methods to remove documents from a collection.

11. **What is replication in MongoDB?**
    - Replication creates multiple copies of data across different servers to provide data redundancy and high availability.

12. **What is sharding in MongoDB?**
    - Sharding is a method of distributing data across multiple servers to handle large datasets and high query loads.

13. **What is the aggregation framework in MongoDB?**
    - A pipeline-based system for processing and transforming documents, used for tasks like data summarization and analysis.

14. **What are the advantages of using MongoDB for web applications?**
    - Flexibility with unstructured data, easy scaling, and efficient handling of large datasets.

15. **How do you handle relationships between data in MongoDB?**
    - By embedding documents within other documents (denormalization) or referencing documents using ObjectIds.

### **Backend Concepts for MERN Stack (15 Questions)**

1. **What is REST API?**
   - A REST API is an architectural style for designing networked applications, relying on stateless, client-server communication via HTTP methods.

2. **What is the difference between REST and GraphQL?**
   - REST exposes endpoints for each resource, while GraphQL allows clients to request specific data by defining queries.

3. **What is JSON Web Token (JWT)?**
   - JWT is a compact, URL-safe token used for securely transmitting information between parties, commonly used for authentication.

4. **What is OAuth?**
   - OAuth is an open standard for authorization, allowing users to grant third-party apps access to their resources without sharing credentials.

5. **What is CORS?**
   - CORS is a security feature that restricts web pages from making requests to a domain other than the one that served the web page.

6. **How do you handle authentication in a MERN stack app?**
   - By using JWT for token-based authentication or services like OAuth for third-party login (e.g., Google, Facebook).

7. **How do you secure API routes in a Node.js application?**
   - By implementing authentication (e.g., JWT) and authorization, using HTTPS, and validating input data.

8. **What is middleware in backend development?**
   - Middleware is software that sits between the request and response to handle tasks like logging, authentication, and error handling.

9. **What is role-based access control (RBAC)?**
   - RBAC is an approach to restricting access based on a user's role within an organization.

10.

 **How do you implement pagination in a Node.js app with MongoDB?**
    - By using query parameters (`limit`, `skip`) in MongoDB queries to limit the number of results returned.

11. **What is the difference between authentication and authorization?**
    - Authentication verifies the user’s identity, while authorization determines the level of access the user has.

12. **What is the purpose of using environment variables in a backend app?**
    - Environment variables store configuration settings, such as database credentials, API keys, and secret tokens.

13. **What is a WebSocket and how is it different from HTTP?**
    - WebSockets provide full-duplex communication channels over a single, long-lived connection, while HTTP is a stateless, request-response protocol.

14. **How do you handle file uploads in a backend application?**
    - By using middleware like `multer` in Node.js to process and save files uploaded by clients.

15. **What are the key differences between monolithic and microservices architectures?**
    - Monolithic architecture builds the entire app as a single unit, while microservices break down the app into smaller, independently deployable services.
